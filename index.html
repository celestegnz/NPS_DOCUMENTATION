Step #1 -> WAIT

En este paso, se define el periodo de espera para que las tablas necesarias estén actualizadas. Este paso es crucial para garantizar que los datos estén completamente cargados antes de continuar con el procesamiento.

Step #2 -> Query Segmentación

En esta sección se describe el proceso de segmentación de los datos utilizados para la muestra NPS. El código de segmentación permite organizar los datos según diferentes criterios, facilitando su posterior análisis. Este proceso ayuda a dividir los datos en grupos, lo que facilita el análisis de cada grupo de forma independiente y proporciona información más detallada.

Step #3 -> Muestreo

El código de muestreo es responsable de organizar y seleccionar un subconjunto de datos representativo, con el fin de obtener una muestra que refleje con precisión las características generales de un conjunto de datos más grande. Este proceso es clave para realizar análisis eficaces sin necesidad de procesar toda la información disponible, lo que podría ser costoso o innecesario.


query_param =f""" 
SELECT * FROM meli-bi-data.WHOWNER.LK_CX_NPS_PARAMETRIZACION WHERE UNIVERSE = '{universe}'
"""
    
universe_params = bq_gaia.query_to_df(query=query_param)
    
e_code_columns = universe_params['E_CODE_COLUMNS'].str.cat(sep=',').split(',')
concat_expression = "CONCAT(" + ", '_', ".join([f"COALESCE({col}, '')" for col in e_code_columns]) + ")"
    
query = f"""
    CREATE OR REPLACE TABLE {sample_table} AS (
    WITH 
    {concat_expression} AS PREV_ECODE,
    [...]
    SELECT
        E.SIT_SITE_ID,
        PREV_ECODE as E_CODE,
    [...] -> resto de columnas
    FROM STG_CX_NPS_PRECAP E
    LEFT JOIN meli-bi-data.WHOWNER.LK_CX_NPS_PARAMETRIZACION P
        ON E.PREV_ECODE = P.E_CODE 
        AND P.UNIVERSE_CAPS = '{universe}'
    LEFT JOIN HISTORICAL DW 
        ON E.CUS_CUST_ID = DW.CUS_CUST_ID 
        AND DW.SAMPLE_DATE < DATE_SUB(CURRENT_DATE(), INTERVAL CAST(P.SURVEY_FREQUENCY AS INT64) DAY)
    INNER JOIN meli-bi-data.WHOWNER.LK_KYC_VAULT_USER v on E.CUS_CUST_ID = v.cus_cust_id
        WHERE 
        RN<= CAP
    );
    """
bq_gaia.execute_query(query=query)

Preguntas Frecuentes (FAQs)

¿Qué es la segmentación?

Consiste en dividir un conjunto de datos en grupos o categorías basados en criterios específicos. Su objetivo es permitir un análisis más enfocado y detallado de cada segmento.

¿Qué es el muestreo?

Proceso de seleccionar una muestra representativa de un conjunto de datos más grande para realizar análisis.

¿Por qué se realizó la migración?

Migrar datos de un universo de un sandbox a un datamesh implica mover los datos hacia una arquitectura descentralizada, donde los dominios responsables de los datos gestionan y optimizan su propio almacenamiento y acceso. El objetivo de esta migración es permitir que cada equipo o unidad dentro de la organización controle sus propios datos de manera más autónoma, al mismo tiempo que se mejora la escalabilidad y la eficiencia, y se facilita el acceso a los datos de forma segura y en tiempo real.

¿Cómo acceder y utilizar el sistema de tickets?

1. ¿Qué es un Ticket de Incidencia?

Un ticket de incidencia se utiliza para reportar problemas o fallos en el sistema, plataforma o procesos. Por ejemplo, si hay errores en un dashboard, un dato incorrecto, o cualquier inconveniente que afecte la operación.

2. ¿Qué es un Ticket de Solicitud?

Un ticket de solicitud se utiliza cuando se requieren cambios en elementos ya cerrados o configuraciones anteriores, como modificaciones de dashboards o mejoras que no son urgentes pero sí necesarias.
